# CFMail 强诊断日志 Design

## 背景
当前 Step 4/7 在 CFMail 轮询超时时，只能看到 attempt 计数，无法判断问题到底出在：
1. 没收到邮件
2. 收到了非目标邮件
3. 收到了 OpenAI 邮件但没有提取出验证码

## 目标
在不改变现有流程行为的前提下，为 CFMail 轮询增加低侵入、可操作的诊断日志，帮助快速判断问题落点。

## 方案
1. 在 `shared/cfmail-utils.js` 增加邮件诊断摘要函数。
2. `background.js:pollCfmailCode()` 每轮抓取邮件后输出摘要日志：
   - 邮件总数
   - 最多 2 封最新邮件的摘要
   - 每封摘要包含：收件人、是否匹配当前邮箱、是否包含 openai/chatgpt 关键词、是否提取到验证码、预览片段
3. 空邮箱时输出明确日志：`mailbox empty`
4. 有邮件但无可用验证码时输出摘要，便于区分“没收到”和“提取失败”

## 日志约束
- 仅截断预览，不输出完整邮件正文
- 如果已提取到验证码，仍走现有成功日志，不额外改变主流程
- 不修改轮询次数、间隔、过滤逻辑

## 验证
- 自动化测试覆盖：
  - 空邮箱时输出 `mailbox empty`
  - 有 OpenAI 邮件但无验证码时输出 `openai=yes` 且 `code=no`
- 语法检查 + 全量测试
